# 05 版权和资源上传说明

这个仓库里混合了三类资料：

1. 自己整理的 Markdown、脚本、复习索引。
2. 课程资料：PPT、试卷、作业、实验说明、同学笔记。
3. 第三方教材 PDF 和教材切片。

## 公开 GitHub 的建议

公开仓库中优先上传第 1 类资料。第 2、3 类资料是否能上传，取决于课程、学校、出版社和原作者授权。没有明确授权时，不建议公开提交。

## 当前私有仓库策略

- `.gitattributes` 为 PDF/PPTX/DOC/DOCX/ZIP 配置了 Git LFS，适合私有仓库或有授权的资料仓库。
- 根目录 AIMA 原书 PDF 和超过 100MB 的概率 PPT 已按私有仓库需求通过 Git LFS 管理。
- `.gitignore` 仍排除了课程压缩包，避免重复上传整包资料。
- `.gitignore` 仍排除了实验生成的可执行文件和临时目录。

## 本地完整资料版

如果你只是本地复习或私有备份，可以保留完整资料。另一个 Codex 接手时，应通过 `docs/02_资料索引.md` 查找这些本地路径。

## 公开精简版

如果你要做公开分享，建议只保留：

- 复习路线
- 考点地图
- 资料索引，不附原文件
- 自己写的作业/实验总结
- 外部链接和引用说明

这样仍然能让仓库成为可用的复习导航，同时降低版权和平台限制风险。
