说明：第 3 题中的原图带有回路 `A-B`。为了把题目写成有限搜索树，以下默认“当前路径上的祖先结点不再重复展开”；同一层或评价值相同的结点，按图中从左到右处理，若左右关系不明显，则按字母序处理。

## 第1题 在线电话翻译系统的 PEAS 分析

这个系统的任务是把英语和日语的电话对话实时翻译给对方，因此可以按 PEAS 分成下面四部分。

| 项目 | 内容 |
| --- | --- |
| `P`，性能度量 | 翻译是否准确，语义是否完整；系统延迟是否足够低；语音识别和语音合成质量是否稳定；噪声和网络波动下是否还能正常工作；用户是否需要频繁纠错；隐私是否得到保护 |
| `E`，环境 | 电话通话场景中的两位用户；英语和日语语音流；网络传输环境；麦克风、扬声器、手机或电脑；噪声、口音、语速变化、停顿、打断等真实情况 |
| `A`，执行器 | 播放翻译后的语音；在界面显示识别文本和翻译文本；必要时提示用户重复、确认或纠正 |
| `S`，感知器 | 麦克风输入；网络接收到的语音数据；用户的手动纠错；当前对话上下文、说话人切换信息、网络状态等 |

环境属性可以进一步分析为：

1. 这是**部分可观测环境**。系统只能接收到语音信号，无法直接获得说话者的真实意图、背景知识和情绪状态。
2. 这是**随机环境**。噪声、口音、语速变化、丢包和延迟都会影响识别和翻译结果。
3. 这是**序贯环境**。当前一句话的翻译往往依赖前文，不同轮对话之间存在明显上下文关系。
4. 这是**动态环境**。用户持续说话，系统必须边接收边处理，不能等环境静止后再决策。
5. 这是**连续环境**。语音信号、时间、音量和语速本身都是连续变化的。
6. 这是**多智能体环境**。至少有两位真实用户在交互，系统的行为会影响对话双方。
7. 从建模角度看，它还是一个**实时任务环境**，因此系统除了“翻得对”，还必须“翻得快”。

## 第2题 医疗诊断系统最合适的 agent 类型

我认为这里最合适的是**基于效用的 agent**。

原因在于，医疗诊断系统面对的不是单纯的“把病名猜出来”，而是要在多个可行方案之间做权衡。系统既要考虑诊断准确率，也要考虑漏诊风险、误诊代价、检查成本、治疗副作用、病情紧急程度等因素。不同决策带来的后果显然不一样，因此最自然的标准不是“是否达到目标”，而是“哪一种决策的综合效果最好”。

例如，对疑似重症患者，系统往往更愿意增加检查、降低漏诊概率；而对普通常见病，则可能更倾向于代价较低、侵入性较小的检查和治疗方案。也就是说，系统需要比较不同结果的优劣，而不是只判断“能不能达到诊断目标”。

其他几类 agent 都不如它合适：

1. **简单反射 agent** 只根据当前输入直接给出反应，适合规则简单的场景，不适合医疗诊断这种高不确定、高风险任务。
2. **基于模型的 agent** 比简单反射强，因为它会维护病人的内部状态和病史，但它本身还不能解决“多个方案之间如何取舍”的问题。
3. **基于目标的 agent** 只关心是否达到某个目标，例如“查明病因”或“治好疾病”，但它难以细致表达不同决策的风险、收益和代价差异。
4. **基于效用的 agent** 能把成功率、风险、花费、副作用等因素统一进评价函数，更符合医疗系统的真实需求。

所以，如果题目要求在四种类型中选一个，医疗诊断系统最适合归为**基于效用的 agent**。严格来说，实际系统通常还会结合内部模型，因此可以理解为“以模型为基础、以效用为决策标准”的 agent。

## 第3题 搜索树、搜索过程与算法分析

先把图写成搜索树。根据题图，状态转移关系可以理解为：

- `S` 的后继为 `A，B`
- `A` 的后继为 `B，D`
- `B` 的后继为 `A，C，D`
- `D` 的后继为 `G`

在“不重复展开本路径上的祖先状态”这一约定下，可得到下列搜索树：

```text
S
├─ A
│  ├─ B
│  │  ├─ C
│  │  └─ D
│  │     └─ G
│  └─ D
│     ├─ B
│     │  └─ C
│     └─ G
└─ B
   ├─ A
   │  └─ D
   │     └─ G
   ├─ C
   └─ D
      ├─ A
      └─ G
```

图中的数字看作启发式函数 `h(n)`，即：

- `h(S)=11`
- `h(A)=8`
- `h(B)=9`
- `h(C)=10`
- `h(D)=5`
- `h(G)=0`

### 3.1 深度优先搜索

深度优先按照“能往下就一直往下”的原则进行。按本题约定，其扩展顺序可记为：

`S, A, B, C, 回溯, D, G`

因此，DFS 找到的解路径为：

`S -> A -> B -> D -> G`

### 3.2 宽度优先搜索

宽度优先逐层搜索。各层如下：

- 第 0 层：`S`
- 第 1 层：`A, B`
- 第 2 层：`A/B, A/D, B/A, B/C, B/D`
- 第 3 层首次出现目标：`A/D/G`

所以 BFS 找到的最浅解为：

`S -> A -> D -> G`

### 3.3 爬山法

爬山法每一步只看当前结点的邻居，选择 `h` 值更小的那个。

1. 在 `S`，比较 `A(8)` 和 `B(9)`，选 `A`
2. 在 `A`，比较 `B(9)` 和 `D(5)`，选 `D`
3. 在 `D`，比较 `B(9)` 和 `G(0)`，选 `G`

所以爬山法的路径为：

`S -> A -> D -> G`

### 3.4 贪婪最佳优先搜索

贪婪最佳优先不是只看当前结点的邻居，而是每次从整个 OPEN 表中选 `h` 最小的结点。

1. 初始只有 `S(11)`
2. 展开 `S` 后，OPEN=`{A(8), B(9)}`，选 `A`
3. 展开 `A` 后，OPEN=`{B(9), B(9), D(5)}`，选 `D`
4. 展开 `D` 后，OPEN 中出现 `G(0)`，因此下一步选 `G`

所以贪婪最佳优先搜索得到：

`S -> A -> D -> G`

### 3.5 四种算法的性质

设 `b` 为分支因子，`d` 为最浅目标深度，`m` 为最大深度。

| 算法 | 本题得到的路径 | 完备性 | 最优性 | 时间复杂度 | 空间复杂度 |
| --- | --- | --- | --- | --- | --- |
| DFS | `S-A-B-D-G` | 一般不完备；若状态空间有限且做环检测，则可完备 | 不最优 | `O(b^m)` | `O(bm)` |
| BFS | `S-A-D-G` | 完备 | 当每步代价相同时最优 | `O(b^{d+1})` | `O(b^{d+1})` |
| 爬山法 | `S-A-D-G` | 不完备 | 不最优 | 一般不保证，常粗略写作 `O(bm)` | `O(1)` |
| 贪婪最佳优先 | `S-A-D-G` | 一般不完备 | 不最优 | 最坏 `O(b^m)` | 最坏 `O(b^m)` |

说明：

1. 本题里爬山法和贪婪搜索都恰好找到了一条不错的路径，但这只是因为启发式值比较“顺”，不能据此推出它们一般都有最优性。
2. BFS 在本题里最稳，因为题图没有给边代价，通常默认每步代价相同，这时 BFS 返回的就是步数最少的解。

## 第4题 计算下一步应展开的叶子结点

题目已经说明这是“一棵部分展开的搜索树”，因此可以认为结点 `A`、`B` 已经被展开，当前 OPEN 中的叶子结点为：

- `C`，路径为 `A -> C`
- `D`，路径为 `A -> D`
- `E`，路径为 `A -> B -> E`
- `F`，路径为 `A -> B -> F`
- `G`，路径为 `A -> B -> G`
- `H`，路径为 `A -> B -> H`

其中已展开顺序为：

`A -> B`

边代价与启发式值整理如下：

| 叶子结点 | 路径 | `g(n)` | `h(n)` |
| --- | --- | --- | --- |
| C | `A -> C` | 19 | 5 |
| D | `A -> D` | 5 | 13 |
| E | `A -> B -> E` | 6 | 10 |
| F | `A -> B -> F` | 8 | 12 |
| G | `A -> B -> G` | 9 | 8 |
| H | `A -> B -> H` | 9 | 10 |

### 4.1 贪婪最佳优先搜索

贪婪最佳优先只比较 `h(n)`。  
当前各叶子结点的启发式值分别为：

- `h(C)=5`
- `h(D)=13`
- `h(E)=10`
- `h(F)=12`
- `h(G)=8`
- `h(H)=10`

其中最小的是 `5`，对应结点 `C`。

所以：

- 扩展前节点顺序：`A -> B`
- 下一步应展开的叶子结点：`C`

### 4.2 一致代价搜索

一致代价搜索只比较从根到当前叶子的真实代价 `g(n)`。  
当前各叶子结点的路径代价为：

- `g(C)=19`
- `g(D)=5`
- `g(E)=6`
- `g(F)=8`
- `g(G)=9`
- `g(H)=9`

最小的是 `5`，对应结点 `D`。

所以：

- 扩展前节点顺序：`A -> B`
- 下一步应展开的叶子结点：`D`

### 4.3 A* 树搜索

A* 使用评价函数：

`f(n)=g(n)+h(n)`

逐个计算得到：

- `f(C)=19+5=24`
- `f(D)=5+13=18`
- `f(E)=6+10=16`
- `f(F)=8+12=20`
- `f(G)=9+8=17`
- `f(H)=9+10=19`

最小的是 `16`，对应结点 `E`。

所以：

- 扩展前节点顺序：`A -> B`
- 下一步应展开的叶子结点：`E`

### 4.4 三种算法的完备性和最优性

| 算法 | 完备性 | 最优性 |
| --- | --- | --- |
| 贪婪最佳优先搜索 | 一般不完备；在有限状态空间并且有判重时可完备 | 不最优 |
| 一致代价搜索 | 当每步代价均为正时完备 | 最优 |
| A* 树搜索 | 当步代价为正且启发式可采纳时完备 | 当启发式可采纳时最优 |

简单比较一下：

1. 贪婪搜索只看 `h(n)`，容易被“看起来很近”的结点误导。
2. UCS 只看真实代价 `g(n)`，因此能保证代价最小解。
3. A* 同时利用 `g(n)` 和 `h(n)`，通常比前两者更高效，也更均衡。

## 第5题 证明：一致性一定推出可采纳性

设启发式函数 `h` 满足：

1. `h(G)=0`，其中 `G` 为目标状态；
2. 对任意相邻结点 `n` 和 `n'`，都有

`h(n) <= c(n,n') + h(n')`

这就是“一致性”条件。

现在要证明：对任意结点 `n`，都有

`h(n) <= h*(n)`

其中 `h*(n)` 表示从 `n` 到目标状态的真实最小代价。也就是说，要证明 `h` 是可采纳的。

取任意一个结点 `n`，设从 `n` 到目标结点 `G` 的最优路径为：

`n=n0, n1, n2, ..., nk=G`

根据一致性，沿这条路径上的每一条边都有：

`h(n0) <= c(n0,n1) + h(n1)`

`h(n1) <= c(n1,n2) + h(n2)`

`...`

`h(nk-1) <= c(nk-1,nk) + h(nk)`

将这些不等式依次展开，可得：

`h(n0) <= c(n0,n1) + c(n1,n2) + ... + c(nk-1,nk) + h(nk)`

由于 `nk=G`，而且 `h(G)=0`，所以上式变成：

`h(n0) <= c(n0,n1) + c(n1,n2) + ... + c(nk-1,nk)`

右边正好就是从 `n` 到目标结点 `G` 的这条最优路径的总代价，也就是 `h*(n)`。因此有：

`h(n) <= h*(n)`

这说明 `h` 从不高估到目标的真实最小代价，所以 `h` 是可采纳的。

结论：只要启发式函数满足一致性，并且目标结点的启发式值为 0，那么它一定是可采纳的。
