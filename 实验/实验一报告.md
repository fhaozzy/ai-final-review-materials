## 五、实验原理

### 1. A* 算法基本思想

A* 算法是一种典型的有信息搜索算法。它在搜索过程中，不仅考虑从起点到当前结点已经付出的实际代价，还考虑从当前结点到目标结点的估计代价。其评价函数定义为：

```text
f(n) = g(n) + h(n)
```

其中：

- `g(n)` 表示从起点到当前结点 `n` 的实际路径代价。
- `h(n)` 表示从当前结点 `n` 到目标结点的启发式估计代价。
- `f(n)` 表示经过当前结点到达目标的综合估计代价。

A* 算法每次从开放表 `openList` 中选取 `f` 值最小的结点进行扩展，因此相比宽度优先搜索、深度优先搜索等无信息搜索方法，能够更快地逼近目标结点。

### 2. 罗马尼亚问题中的启发函数

在罗马尼亚地图问题中，实验给出的数组 `h[20]` 表示各城市到目标城市的启发值。目标城市选择为 `B`，即布加勒斯特（Bucharest）。这些启发值可看作城市到目标城市的估计距离(即为直线距离)，因此适合作为 A* 算法中的 `h(n)`。

### 3. 数据结构设计

本实验中主要使用了如下数据结构：

- `graph[20][20]`：邻接矩阵，用于存储城市之间的路径代价。
- `openList`：开放表，保存当前待扩展的结点。
- `closeList`：关闭表，保存已经扩展过的结点。
- `parent[20]`：父结点数组，用于记录路径，便于最终回溯得到最优解。
- `node` 结构体：保存结点编号、`g` 值、`h` 值和 `f` 值。

## 六、实验方法与步骤

### 1. 创建罗马尼亚地图搜索图

首先构建图结构 `Graph`，使用邻接矩阵保存各城市之间的连通关系和路径代价。图中共设置 20 个结点，并按照罗马尼亚问题中的道路连接关系初始化边权。

### 2. 初始化搜索信息

搜索开始前，需要完成以下初始化：

1. 将起始城市作为源结点加入 `openList`。
2. 将所有访问标记数组置为初值。
3. 将 `parent` 数组初始化为 `-1`，表示尚无父结点。

### 3. 执行 A* 搜索

搜索过程中执行以下步骤：

1. 从 `openList` 中取出 `f` 值最小的结点。
2. 判断该结点是否为目标结点，若是则结束搜索。
3. 否则扩展该结点的所有邻接结点。
4. 若邻接结点未被访问，则加入开放表，并记录其父结点。
5. 若邻接结点已在开放表中，但通过当前路径得到的 `g` 值更小，则更新该结点的 `g` 值、`f` 值以及父结点。
6. 重复上述步骤，直到找到目标结点或开放表为空。

### 4. 输出最优路径

当搜索结束后，根据 `parent` 数组从目标结点逆向回溯到起点，并借助栈结构逆序输出最终路径，同时统计总代价。

## 七、搜索过程分析

以从 `A` 到 `B` 的路径搜索为例，A* 算法的主要搜索过程如下：

1. 初始结点为 `A`，其 `g(A)=0`，`h(A)=366`，因此 `f(A)=366`。
2. 扩展 `A` 后，可到达 `Z`、`S`、`T`，对应的 `f` 值分别为：
   - `Z`：`g=75`，`h=374`，`f=449`
   - `S`：`g=140`，`h=253`，`f=393`
   - `T`：`g=118`，`h=329`，`f=447`
3. 由于 `S` 的 `f` 值最小，因此优先扩展 `S`。
4. 扩展 `S` 后得到 `R`、`F`、`O` 等结点，其中：
   - `R`：`g=220`，`h=193`，`f=413`
   - `F`：`g=239`，`h=178`，`f=417`
5. 接着优先扩展 `R`，再扩展 `P`，最终到达目标结点 `B`。

因此，A* 算法找到的最优路径为：

```text
A -> S -> R -> P -> B
```

总代价为：

```text
140 + 80 + 97 + 101 = 418
```

## 八、源程序

```cpp
#include<iostream>
#include<vector>
#include<memory.h>
#include<stack>
#include<algorithm>
#define A 0
#define B 1
#define C 2
#define D 3
#define E 4
#define F 5
#define G 6
#define H 7
#define I 8
#define L 9
#define M 10
#define N 11
#define O 12
#define P 13
#define R 14
#define S 15
#define T 16
#define U 17
#define V 18
#define Z 19

using namespace std;

int h[20] =
{ 366,0,160,242,161,
178,77,151,226,244,
241,234,380,98,193,
253,329,80,199,374 };

struct node
{
    int g;
    int h;
    int f;
    int name;
    node(int name, int g, int h)
    {
        this->name = name;
        this->g = g;
        this->h = h;
        this->f = g + h;
    };
    bool operator <(const node &a) const
    {
        return f < a.f;
    }
};

class Graph
{
public:
    Graph()
    {
        memset(graph, -1, sizeof(graph));
    }
    int getEdge(int from, int to)
    {
        return graph[from][to];
    }
    void addEdge(int from, int to, int cost)
    {
        if (from >= 20 || from < 0 || to >= 20 || to < 0)
            return;
        graph[from][to] = cost;
    }

    void init()
    {
        addEdge(O, Z, 71);
        addEdge(Z, O, 71);

        addEdge(O, S, 151);
        addEdge(S, O, 151);

        addEdge(Z, A, 75);
        addEdge(A, Z, 75);

        addEdge(A, S, 140);
        addEdge(S, A, 140);

        addEdge(A, T, 118);
        addEdge(T, A, 118);

        addEdge(T, L, 111);
        addEdge(L, T, 111);

        addEdge(L, M, 70);
        addEdge(M, L, 70);

        addEdge(M, D, 75);
        addEdge(D, M, 75);

        addEdge(D, C, 120);
        addEdge(C, D, 120);

        addEdge(C, R, 146);
        addEdge(R, C, 146);

        addEdge(S, R, 80);
        addEdge(R, S, 80);

        addEdge(S, F, 99);
        addEdge(F, S, 99);

        addEdge(F, B, 211);
        addEdge(B, F, 211);

        addEdge(P, C, 138);
        addEdge(C, P, 138);

        addEdge(R, P, 97);
        addEdge(P, R, 97);

        addEdge(P, B, 101);
        addEdge(B, P, 101);

        addEdge(B, G, 90);
        addEdge(G, B, 90);

        addEdge(B, U, 85);
        addEdge(U, B, 85);

        addEdge(U, H, 98);
        addEdge(H, U, 98);

        addEdge(H, E, 86);
        addEdge(E, H, 86);

        addEdge(U, V, 142);
        addEdge(V, U, 142);

        addEdge(I, V, 92);
        addEdge(V, I, 92);

        addEdge(I, N, 87);
        addEdge(N, I, 87);
    }

private:
    int graph[20][20];
};

bool list[20];
vector<node> openList;
bool closeList[20];
stack<int> road;
int parent[20];

void A_star(int goal, node &src, Graph &graph)
{
    openList.push_back(src);
    list[src.name] = true;
    sort(openList.begin(), openList.end());

    while (!openList.empty())
    {
        if (openList[0].name == goal)
            break;

        node cur = openList[0];
        openList.erase(openList.begin());
        list[cur.name] = false;
        closeList[cur.name] = true;

        for (int i = 0; i < 20; i++)
        {
            int cost = graph.getEdge(cur.name, i);
            if (cost == -1 || closeList[i])
                continue;

            int newG = cur.g + cost;

            if (!list[i])
            {
                openList.push_back(node(i, newG, h[i]));
                list[i] = true;
                parent[i] = cur.name;
            }
            else
            {
                for (int j = 0; j < openList.size(); j++)
                {
                    if (openList[j].name == i && newG < openList[j].g)
                    {
                        openList[j].g = newG;
                        openList[j].f = openList[j].g + openList[j].h;
                        parent[i] = cur.name;
                        break;
                    }
                }
            }
        }
        sort(openList.begin(), openList.end());
    }
}

void print_result(Graph &graph)
{
    int p = openList[0].name;
    int lastNodeNum;
    road.push(p);
    while (parent[p] != -1)
    {
        road.push(parent[p]);
        p = parent[p];
    }
    lastNodeNum = road.top();
    int cost = 0;
    cout << "solution: ";
    while (!road.empty())
    {
        cout << road.top() << "-> ";
        if (road.top() != lastNodeNum)
        {
            cost += graph.getEdge(lastNodeNum, road.top());
            lastNodeNum = road.top();
        }
        road.pop();
    }
    cout << "end" << endl;
    cout << "cost:" << cost << endl;
}

int main()
{
    memset(list, false, sizeof(list));
    memset(closeList, false, sizeof(closeList));
    memset(parent, -1, sizeof(parent));

    Graph graph;
    graph.init();

    node src(A, 0, h[A]);
    A_star(B, src, graph);
    print_result(graph);

    return 0;
}
```

## 十、时间复杂度分析

A* 算法的最坏情况下时间复杂度通常可表示为：

```text
O(b^d)
```

其中：

- `b` 表示搜索树的分支因子。
- `d` 表示目标结点所在的深度。

其原因是：在最坏情况下，A* 仍可能扩展大量结点，甚至接近遍历整棵搜索树。

本实验中还使用了 `sort(openList.begin(), openList.end())` 对开放表排序。若开放表长度为 `n`，每次排序的复杂度约为 `O(n log n)`。因此程序的实际运行开销还与开放表大小有关。

空间复杂度方面，A* 需要保存开放表和关闭表中的结点信息，因此空间复杂度也较高，通常也接近指数级。相比深度优先搜索，A* 更占用内存，但能够更快地找到最优解。

## 十一、实验结果分析

通过本次实验可以看出，A* 算法通过综合考虑实际路径代价和启发式估计代价，使搜索过程具有明确方向性。与无信息搜索相比，A* 不会盲目扩展大量无关结点，因此能够更高效地找到目标路径。

在罗马尼亚地图问题中，启发函数较为合理，因此算法能够快速收敛到最优解。实验结果表明，A* 算法不仅能够正确输出路径，还能够保证总代价最小，说明实现结果正确。

## 十二、思考题

### 1. 宽度优先搜索、深度优先搜索、一致代价搜索、迭代加深深度优先搜索哪种方法最优

这几种方法各有适用场景，不能简单说某一种绝对最优。

- 宽度优先搜索在单位代价条件下可以得到最短路径，但空间开销较大。
- 深度优先搜索空间开销小，但不保证最优解，且可能陷入较深无效路径。
- 一致代价搜索在边权不同时能够保证得到最优解，因此在一般代价搜索问题中更优。
- 迭代加深深度优先搜索兼具深度优先搜索的低空间开销和宽度优先搜索的完备性，在单位代价情况下效果较好。

综合来看：

- 若要求严格最优解且边权不同，则一致代价搜索更优。
- 若边权相同且空间受限，则迭代加深搜索较优。

### 2. 贪婪最佳优先搜索和 A* 搜索哪种方法最优

贪婪最佳优先搜索只考虑 `h(n)`，即只关注“离目标看起来有多近”，因此搜索速度可能较快，但不一定能找到最优解。

A* 搜索同时考虑 `g(n)` 和 `h(n)`，只要启发函数满足可采纳性，就能够保证最优解。因此从求解质量来看，A* 更优；从某些场景下的搜索速度来看，贪婪搜索可能更快，但稳定性和最优性不如 A*。

### 3. 分析比较无信息搜索策略和有信息搜索策略

无信息搜索策略不利用任何关于目标位置的先验知识，只根据问题定义进行系统搜索，例如宽度优先搜索、深度优先搜索和一致代价搜索。其优点是通用性强、实现简单；缺点是搜索范围大、效率较低。

有信息搜索策略利用启发式函数估计当前状态与目标状态之间的距离，例如贪婪最佳优先搜索和 A* 搜索。其优点是搜索方向明确、效率高；缺点是搜索效果依赖启发函数的质量，若启发信息不合理，则可能影响搜索结果。

因此，在能够构造较好启发函数的情况下，有信息搜索策略通常优于无信息搜索策略。
