## 五、实验原理

### 1. 约束满足问题概述

约束满足问题（Constraint Satisfaction Problem，CSP）是人工智能中的一类经典问题。其核心思想是：在给定变量集合、变量取值范围和约束条件的前提下，寻找满足全部约束的变量赋值方案。

一个 CSP 通常由以下三部分组成：

- 变量集合：需要求值的对象。
- 取值域：每个变量允许取的值。
- 约束集合：变量之间必须满足的条件。

### 2. 八皇后问题的 CSP 建模

八皇后问题要求在 `8×8` 的国际象棋棋盘上放置 8 个皇后，使得任意两个皇后都不能互相攻击。由于皇后能够攻击同一行、同一列以及同一对角线上的其他皇后，因此需要满足以下约束：

1. 任意两个皇后不能在同一列。
2. 任意两个皇后不能在同一主对角线。
3. 任意两个皇后不能在同一副对角线。

本实验中采用“每一行放一个皇后”的方式进行建模，因此：

- 变量：第 `i` 行皇后的位置。
- 取值域：第 `i` 行皇后可以放在第 `1~8` 列中的任意一列。
- 约束：
  - 不同行的皇后不能在同一列。
  - 不同行的皇后不能在同一条对角线上。

### 3. 回溯搜索算法思想

回溯搜索是一种深度优先的搜索方法。其基本思想是：

1. 从第一个变量开始赋值。
2. 若当前赋值满足约束，则继续为下一个变量赋值。
3. 若当前赋值不满足约束，则撤销当前赋值，尝试其他可能取值。
4. 当所有变量均完成赋值时，即得到一个可行解。

在八皇后问题中，回溯搜索的具体过程为：

1. 从第 1 行开始，尝试将皇后放在某一列。
2. 判断当前列以及两条对角线是否已被占用。
3. 若合法，则递归处理下一行。
4. 若当前行没有合法位置，则回溯到上一行，重新尝试。

### 4. 关键数组含义

本实验程序中使用了以下数组：

- `a[i]`：表示第 `i` 行皇后放置的列号。
- `b[j]`：表示第 `j` 列是否已有皇后。
- `c[i+j]`：表示主对角线是否已有皇后。
- `d[i-j+7]`：表示副对角线是否已有皇后。

通过这几个数组，可以快速判断某个位置是否可以放置皇后，从而提高搜索效率。

## 六、实验方法与步骤

### 1. 初始化数据结构

定义数组 `a`、`b`、`c`、`d` 和计数变量 `sum`。其中：

- `a` 用于保存每一行皇后的摆放结果。
- `b`、`c`、`d` 用于标记列和两类对角线是否被占用。
- `sum` 用于统计解的总数。

### 2. 设计递归函数

定义递归函数 `searchh(int i)`，表示当前处理第 `i` 行皇后的放置。

在函数中，依次尝试第 `i` 行的每一列 `j`：

1. 若第 `j` 列、主对角线 `i+j` 和副对角线 `i-j+7` 均未被占用，则说明该位置合法。
2. 将皇后放置在该位置，并更新占用标记。
3. 若当前已经是第 8 行，则找到一组完整解，计数器加 1。
4. 否则递归求解下一行。
5. 递归返回后撤销当前标记，继续尝试其他列。

### 3. 输出结果

从 `searchh(1)` 开始执行递归搜索，最终输出八皇后问题的解的总数。

## 七、源程序

```cpp
#include<iostream>
using namespace std;

int a[9];
int b[9] = {0};
int c[16] = {0};
int d[16] = {0};
int sum = 0;

void searchh(int i)
{
    for(int j = 1; j <= 8; j++)
    {
        if((!b[j]) && (!c[i + j]) && (!d[i - j + 7]))
        {
            a[i] = j;
            b[j] = 1;
            c[i + j] = 1;
            d[i - j + 7] = 1;

            if(i == 8)
                sum++;
            else
                searchh(i + 1);

            b[j] = 0;
            c[i + j] = 0;
            d[i - j + 7] = 0;
        }
    }
}

int main()
{
    searchh(1);
    cout << "sum=" << sum << endl;
    return 0;
}
```

## 八、程序运行结果

程序运行后，输出结果如下：

```text
sum=92
```

这说明八皇后问题共有 `92` 个不同解，与经典结论一致，证明程序实现正确。

## 九、结果分析

本实验采用回溯搜索方法求解八皇后问题。程序从第一行开始，逐行尝试放置皇后，并利用列、主对角线和副对角线的标记数组快速判断某一位置是否满足约束条件。

与穷举法相比，回溯搜索在搜索过程中一旦发现某一步不满足约束，就立即停止继续向下搜索，并返回上一层重新尝试，这种“剪枝”思想有效减少了无效搜索空间，提高了求解效率。

实验结果表明，使用回溯法能够完整求出八皇后问题的全部 92 组解，说明该方法在求解结构化约束满足问题时具有较好的效果。

## 十、时间复杂度分析

### 1. 最坏情况下的复杂度

对于八皇后问题，若不考虑剪枝，8 行皇后在不同列上的排列总数约为：

```text
8! = 40320
```

回溯搜索在最坏情况下需要对大量排列进行检测，因此其时间复杂度通常可近似看作阶乘级，即：

```text
O(N!)
```

对于一般的 `N` 皇后问题，随着 `N` 的增大，搜索规模增长非常快。

### 2. 剪枝带来的优化

本实验在搜索过程中加入了约束检查：

- 列冲突检查
- 主对角线冲突检查
- 副对角线冲突检查

这些约束判断可以在常数时间内完成，因此能够在搜索早期提前排除大量不可能的分支，显著提高求解效率。

### 3. 空间复杂度

程序主要使用递归栈和若干辅助数组保存状态，因此空间复杂度较低。对于 `N` 皇后问题，空间复杂度大致为：

```text
O(N)
```
